# Delta 推論の実験 — 結果

**日付：** 2026-05-19
**問い：** 目が静止した壁を再レンダリングしないように、Atome は再計算を
スキップできるか？ 完全再計算 vs 時間 delta の三値 matvec を測定する。

## セットアップ

- 三値の 256×256 行列（944K Atome モデルの `d_model` を反映）、約 1/3 がゼロ
- 256 ステップの入力ストリーム、3 つの入力レジーム
- `out_new = out_old + W @ (x_new - x_prev)` — 閾値 0 で厳密
- 選択的な `x_prev` 更新：伝播したチャネルのみ更新するため、各チャネルの
  保留誤差は常に `threshold` で有界——これが積分発火（integrate-and-fire）
- エネルギー代理指標：`iters` = 内側ループの周回数（各周回は trit を 1 つ展開して分岐し、
  MAC を行うか否かにかかわらず MCU で約 1 サイクル）。決定的かつ厳密。

## 結果（ホスト、経路ごとのサイクルを個別に測定）

| レジーム | 閾値 | iter 高速化 | サイクル高速化 | 最大誤差 |
|---|---|---|---|---|
| **センサーストリーム** | 0.000 | 1.00× | 1.04× | 0.00001 |
| （相関入力） | 0.005 | **17.52×** | 15.66× | 0.00715 |
| | 0.020 | **51.24×** | 42.96× | 0.01845 |
| | 0.050 | **59.67×** | 49.07× | 0.03455 |
| **トークン埋め込み** | 0.000 | 1.00× | 1.04× | 0.00001 |
| （無相関 / LM 生成） | 0.005 | 1.01× | 1.05× | 0.00072 |
| **隠れ状態の代理** | 0.000 | 3.30× | 3.25× | 0.00001 |
| （約 30 % のチャネルが動く） | 0.005 | 3.39× | 3.34× | 0.00171 |
| | 0.020 | 3.68× | 3.59× | 0.01163 |

QEMU Cortex-M3（`mps2-an385`）：`iters` はホストと**ビット単位で同一**
（16,711,680 / 954,112 / 326,144 / …）——エネルギー代理指標はターゲット ISA 上で
厳密に再現します。ターゲット上の DWT サイクルカウンタは 0 を読みます。QEMU の
`mps2-an385` が `DWT->CYCCNT` をモデル化しないためです。実シリコンのサイクル数には
Cortex-M3 の開発ボードかサイクル精密モデルが必要です。ホストの実時間はすでに
`iters` が実サイクルを追うことを確認しています（15.66× サイクル vs 17.52× iter——
差はループ/呼び出しのオーバーヘッド）。

## 発見

1. **利得は本物で大きい——ただし相関入力に限る。** センサー風の
   ストリームは、閾値 0.005 で**操作数が 17.5× 少なく**動作し、最悪ケースの
   出力誤差は 0.007（重みのスケールは 0.05 なので、これは典型的な logit の
   約 0.7 %）です。閾値 0.02 では **51×** です。サーモスタット、加速度計の
   ジェスチャ検出器、音声キーワード検出器である MCU デバイスにとって、
   これは推論エネルギーの直接的な 17〜51× の削減です。

2. **トークン LM 生成にはただ飯なし——確認済み。** シナリオ B は
   1.0× に留まります。連続するバイト埋め込みは無相関で、スキップすべき
   「静止した壁」がありません。これは誠実な結果で、予測と一致します。Delta
   推論は*入力モダリティ*の最適化であり、万能ではありません。

3. **ネットワーク中段の隠れ状態はその中間（約 3.3×）。** 閾値なしでも、
   ステップごとに約 30 % のチャネルが動く残差ストリームは 3.3× を無償で与えます
   （厳密、誤差 1e-5）——matvec の 70 % が本当に冗長だからです。これは
   最も興味深い数字です：トークン入力が助けにならないときでも、delta 推論が
   ネットワークの*内部*で助けになることを示唆します。特に、状態がゆっくり
   進化する SSM 経路で顕著です。

4. **閾値は文字どおり発火閾値です。** `x_prev` は伝播したチャネルにのみ
   更新されるため、閾値未満のドリフトを持つチャネルは静かに積分し、
   バーを越えると一度発火してリセットします。誤差は `threshold` で有界で、
   蓄積はなく、周期的な「サッカード」リフレッシュも不要です。
   エネルギー/精度のトレードオフは単一のつまみです。

## 誠実な制約

- 合成 256×256 行列は代表的ですが、学習済みの Atome 重みセットではありません
  ——実際の重みのスパース構造は定数を（傾向ではなく）ずらすかもしれません。
- delta 化されるのは matvec のみです。LayerNorm/SSM/アテンションは非線形で、
  完全な統合にはそれらの delta 対応（あるいは周期的リフレッシュ）版が必要です。
- 「iters」は matvec 内側ループの忠実なエネルギー代理指標ですが、
  メモリトラフィックのエネルギーを無視します。これは実 MCU で支配的になり得ます
  ——シリコン上の*実際の*高速化はより高い（データ移動が少ない）か、より低い
  （列優先の delta アクセスパターンによるキャッシュ挙動の悪化）かもしれません。
  開発ボードでの測定が必要です。
- トークンレジームの結果（1.0×）は誠実な上限です：delta 推論を LM 生成の
  高速化として売り込まないでください。ストリーミングセンサー分類器に売り込んでください。

## 推奨

delta 推論を**ストリーミング分類器のデプロイ向けのオプトインモード**として
配線してください（センサー入力での Atome の `atome_classify` パス）——生成パスでは
ありません。SSM 経路は次に拡張するのに自然な場所です——その状態はネットワークで
最も遅い信号です。閾値を（セキュリティスタックの）L11 状態ノルムモニターと組み合わせて
ドリフトの番犬とします。期待される範囲：サーモスタット/音声/ジェスチャ級のデバイスで、
調整可能で有界な精度コストと引き換えに、**推論エネルギーを 15〜50× 削減**。

# 拡張：実際の 944K Atome モデルでの検証

上記の合成実験はランダム行列を使いました。本節は delta パスを
`checkpoints/atome_1m_v1.pt`（実際の学習済み 944K モデル、val_loss 1.0545）に対し、
実際の 196 バイトの TinyStories 一節で走らせます。

`capture_real.py` は各ブロックにフックし、実際の post-norm 入力と
各経路出力を捕捉し、信号ごとの delta 冗長性を測定します。次に `bench_real.c` が、
モデルの**実際の三値化アテンション Wv** を用いて、実際に捕捉されたストリームに対し
**C** の `dm_matvec_delta` を走らせ、C プリミティブが numpy の予測を再現することを
確認します。

## 経路ごとの delta-matvec 高速化（実重み、8 ブロック平均）

| matvec が消費する信号 | 閾値 0.0 | 閾値 0.02 | 閾値 0.05 | 閾値 0.10 |
|---|---|---|---|---|
| post-norm 入力 `h` | 1.00× | 1.05× | 1.11× | 1.17× |
| conv 経路出力 | 1.01× | 1.06× | 1.12× | 1.22× |
| **SSM 経路出力** | 1.00× | **4.06×** | **12.27×** | **45.16×** |
| アテンション経路出力 | 1.02× | 1.07× | 1.15× | 1.27× |

## C プリミティブの相互検証（ブロック 0、実 Wv 256×256、スケール 0.0412）

| matvec | 閾値 0.0 | 閾値 0.02 | 閾値 0.05 | 閾値 0.10 |
|---|---|---|---|---|
| `Wv @ h`（numpy 予測） | 1.03× | 1.26× | 1.53× | 1.61× |
| `Wv @ h`（**C 実測**） | 1.03× | 1.26× | 1.53× | 1.61× |
| `Wv @ ssm_out`（numpy 予測） | 1.00× | 3.12× | 8.60× | 33.64× |
| `Wv @ ssm_out`（**C 実測**） | 1.00× | 3.12× | 8.60× | 33.64× |

C の delta プリミティブと numpy 参照は、実際の学習済み重みと実際の活性化で
**厳密に**一致します。チャネルごとの最大誤差は閾値以下に留まります
（閾値 0.10 で実測 0.10000）——積分発火の限界は実データで成立します。

## 発見——そして 1 つの誠実な否定結果

1. **SSM 経路出力が delta のスイートスポットであり、それも大差で。** 実重みでは
   4〜45× delta 圧縮可能で、ブロック内の他のあらゆる信号は 1.0〜1.3× です。
   SSM 出力に供給される matvec は、チャネルごとに約 5 % の誤差で、閾値 0.05 で
   12× 少ない仕事をします。

2. **SSM 自体は delta 圧縮できない——そしてそれで良い。** それは
   チャネルごとの再帰 `h_t = a·h_{t-1} + b·x_t` で、各ステップが前のステップに
   依存するため、どのステップもスキップできません。しかしそれはすでに O(チャネル数)
   で、ボトルネックではありません。delta 推論におけるその役割は*遅い信号の生成器*です：
   チャネルごとのローパスフィルタなので、その出力はネットワークで最も位置相関の
   高い信号であり——これこそが、その下流の matvec を delta フレンドリーにします。
   以前の RESULTS の推奨（「delta を SSM に拡張せよ」）は半分正しかったのです：
   SSM 再帰そのものではなく、SSM を*消費する* matvec に拡張してください。

3. **誠実な否定結果：post-norm `h`、conv 出力、アテンション出力は delta
   フレンドリーでない（約 1.0〜1.3×）。** LayerNorm は各位置を再正規化するため
   `h` はほぼ全チャネルでずれ、conv とアテンションの出力は位置ごとに本当に変化します。
   （`h` を消費する）アテンション Wq/Wk/Wv 投影での delta 推論はほとんど何も
   もたらしません。そこには配備しないでください。

4. **より深いブロックはブロック 0 より delta フレンドリー**（8 ブロック平均 45×
   vs ブロック 0 の閾値 0.10 で 33×）——SSM 状態は深さとともに温まるため、
   遅い信号の性質はネットワークの深部で強まります。

## 精緻化された推奨

delta 推論を**SSM 経路出力を消費する matvec 層**に配備してください——
アテンション投影でも、SSM 再帰そのものでもありません。実際の 944K モデルでは、
これは閾値 0.05 での**実測 8〜12× の計算削減**（チャネルごとに約 5 % の誤差、有界）で、
閾値 0.10 で 33〜45× に上がります。閾値を L11 状態ノルムモニターと組み合わせて
ドリフトの番犬とします。

## 品質コスト — 測定済み（2026-05-20、`quality_real.py`）

以前の草稿はエネルギー数値を伏せていました。閾値化誤差の*品質*コストが
未測定だったためです。今や測定済みです。信号 S を消費する matvec での delta 推論は、
積分発火の閾値化をした S を厳密な matvec に供給するのと等価です。よって各ブロックの
SSM 出力を閾値化し、実際の 944K モデルの残りを厳密に走らせ、交差エントロピーを
測定します。

| 閾値 | SSM 経路の高速化 | Δ 困惑度 |
|---|---|---|
| 0.00 | 1.0× | +0.00%（厳密——健全性チェック） |
| 0.02 | 4.1× | −0.46%（ノイズの範囲内） |
| **0.05** | **12.6×** | **+0.57%** |
| 0.10 | 49× | +5.6% |
| 0.20 | 320× | +11.5%（破綻） |

**主張は生き残ります。** 閾値 0.05 で、SSM 経路は学習済みモデルにおいて
**+0.57 % の困惑度で 12.6× の反復回数削減**という本物の値を与えます——出荷可能な
トレードオフです。0.10 では攻めすぎ（49× / +5.6 %）、0.20 では破綻します。これはまた、
「SSM 出力はほぼ一定で何も寄与しない」という懸念を反証します：もし何も寄与しないなら、
それを古びさせることはどの閾値でも無料のはずですが——代わりに 0.20 は +11.5 % を
費やすので、SSM 出力は本当に重要で、モデルは*有界な*古さを本当に許容します。

## 残る誠実な但し書き

上記の品質数値は**プリフィル位置の交差エントロピー（prefill）**であり、自己回帰
生成ではありません。SSM のローパス性質は生成に引き継がれるはずですが（再帰
フィルタなので）、生成ステップの測定は実行されていません。12.6×/+0.57 % の数字は、
その但し書きを付けて述べてください。
