# Delta 推理实验 — 结果

**日期：** 2026-05-19
**问题：** Atome 能否像眼睛不重新渲染一面静止的墙那样跳过重新计算？
测量完全重算 vs 时间 delta 三元 matvec。

## 设置

- 256×256 三元矩阵（对应 944K Atome 模型的 `d_model`），约 1/3 为零
- 256 步的输入流，三种输入区间
- `out_new = out_old + W @ (x_new - x_prev)` — 在阈值 0 处精确
- 选择性的 `x_prev` 更新：只有被传播的通道会更新，因此每个通道的
  待处理误差始终被 `threshold` 界定——这就是积分-发放（integrate-and-fire）
- 能量代理：`iters` = 内层循环的次数（每次都会解包一个 trit 并分支，
  在 MCU 上约 1 周期，无论是否做 MAC）。确定且精确。

## 结果（主机，逐路径周期分别测量）

| 区间 | 阈值 | iter 加速 | 周期加速 | 最大误差 |
|---|---|---|---|---|
| **传感器流** | 0.000 | 1.00× | 1.04× | 0.00001 |
| （相关输入） | 0.005 | **17.52×** | 15.66× | 0.00715 |
| | 0.020 | **51.24×** | 42.96× | 0.01845 |
| | 0.050 | **59.67×** | 49.07× | 0.03455 |
| **Token 嵌入** | 0.000 | 1.00× | 1.04× | 0.00001 |
| （不相关 / LM 生成） | 0.005 | 1.01× | 1.05× | 0.00072 |
| **隐藏状态代理** | 0.000 | 3.30× | 3.25× | 0.00001 |
| （约 30 % 通道移动） | 0.005 | 3.39× | 3.34× | 0.00171 |
| | 0.020 | 3.68× | 3.59× | 0.01163 |

QEMU Cortex-M3（`mps2-an385`）：`iters` 与主机**逐位相同**
（16,711,680 / 954,112 / 326,144 / …）——能量代理在目标 ISA 上
精确复现。目标端的 DWT 周期计数器读数为 0，因为 QEMU 的 `mps2-an385`
不建模 `DWT->CYCCNT`；真实硅片的周期数需要一块 Cortex-M3 开发板
或一个逐周期精确的模型。主机的挂钟已经确认 `iters`
跟踪真实周期（15.66× 周期 vs 17.52× iter——差距是循环/调用开销）。

## 发现

1. **收益真实且巨大——但仅对相关输入。** 一个传感器风格的
   流在阈值 0.005 下运行时**操作数少 17.5×**，其最坏情况
   输出误差为 0.007（权重缩放为 0.05，所以这约是一个典型
   logit 的 0.7 %）。在阈值 0.02 下则为 **51×**。对于一个是
   恒温器、加速度计手势检测器或音频关键词检出器的 MCU 设备，
   这是推理能耗上直接 17–51× 的削减。

2. **对 token-LM 生成没有免费午餐——已确认。** 场景 B 保持在
   1.0×。连续的字节嵌入不相关；没有"静止的墙"
   可跳过。这是诚实的结果，且与预测相符。Delta
   推理是一种*输入模态*优化，而非普适优化。

3. **网络中段的隐藏状态介于两者之间（约 3.3×）。** 即便没有阈值，
   一个约 30 % 通道每步移动的残差流也能免费给出 3.3×
   （精确，误差 1e-5），因为 70 % 的 matvec 是真正冗余的。这是
   最有意思的数字：它表明 delta 推理即便在 token 输入无益时也在
   网络*内部*有帮助，尤其是对状态缓慢演化的 SSM 通路。

4. **阈值字面上就是一个发放阈值。** 因为 `x_prev` 只对被
   传播的通道更新，一个漂移低于阈值的通道会静默地积分，
   直到它越过门限、发放一次、然后复位。误差被 `threshold` 界定，
   没有累积，也不需要周期性的"扫视"（saccade）刷新。
   能量/精度的权衡是一个单一旋钮。

## 诚实的局限

- 256×256 的合成矩阵有代表性，但不是训练好的 Atome 权重
  集合——真实权重的稀疏结构可能会移动常量（而非趋势）。
- 只有 matvec 被"delta 化"。LayerNorm/SSM/注意力是非线性的；一次完整
  的集成需要它们的 delta 感知（或周期性刷新）变体。
- "iters"是 matvec 内层循环的忠实能量代理，但忽略了
  内存流量能量，后者在真实 MCU 上可能占主导——硅片上*真实*的
  加速可能更高（更少的数据搬运）或更低（列主序 delta 访问模式带来
  更差的缓存行为）。需要一次开发板上的测量。
- token 区间的结果（1.0×）是诚实的上限：不要把 delta 推理
  宣传为一种 LM 生成加速。请把它宣传给流式传感器分类器。

## 建议

把 delta 推理接线为**面向流式分类器部署的可选（opt-in）模式**
（Atome 在传感器输入上的 `atome_classify` 路径），而非生成路径。
SSM 通路是接下来扩展它的自然之处——其状态是网络中最慢
的信号。把该阈值与 L11 状态范数监视器（来自安全栈）配对
作为漂移看门狗。预期包络：对于恒温器/音频/手势类设备，
**推理能耗降低 15–50×**，代价是一个可调、有界的精度成本。

# 扩展：在真实的 944K Atome 模型上验证

上面的合成实验用的是随机矩阵。本节把 delta 路径
运行在 `checkpoints/atome_1m_v1.pt`（真实训练的 944K 模型，
val_loss 1.0545）上，使用一段真实的 196 字节 TinyStories 文本。

`capture_real.py` 挂钩每个块，捕获真实的 post-norm 输入和
每条通路输出，并测量逐信号的 delta 冗余。随后 `bench_real.c`
使用模型的**真实三元化注意力 Wv** 在真实捕获的流上运行 **C**
的 `dm_matvec_delta`，并确认该 C 原语
复现了 numpy 的预测。

## 逐通路 delta-matvec 加速（真实权重，8 块平均）

| matvec 消费的信号 | 阈值 0.0 | 阈值 0.02 | 阈值 0.05 | 阈值 0.10 |
|---|---|---|---|---|
| post-norm 输入 `h` | 1.00× | 1.05× | 1.11× | 1.17× |
| conv 通路输出 | 1.01× | 1.06× | 1.12× | 1.22× |
| **SSM 通路输出** | 1.00× | **4.06×** | **12.27×** | **45.16×** |
| 注意力通路输出 | 1.02× | 1.07× | 1.15× | 1.27× |

## C 原语交叉核对（块 0，真实 Wv 256×256，缩放 0.0412）

| matvec | 阈值 0.0 | 阈值 0.02 | 阈值 0.05 | 阈值 0.10 |
|---|---|---|---|---|
| `Wv @ h`（numpy 预测） | 1.03× | 1.26× | 1.53× | 1.61× |
| `Wv @ h`（**C 实测**） | 1.03× | 1.26× | 1.53× | 1.61× |
| `Wv @ ssm_out`（numpy 预测） | 1.00× | 3.12× | 8.60× | 33.64× |
| `Wv @ ssm_out`（**C 实测**） | 1.00× | 3.12× | 8.60× | 33.64× |

C delta 原语与 numpy 参考在真实训练权重和真实激活上
**精确**一致。每通道最大误差保持 ≤ 阈值
（在阈值 0.10 处实测 0.10000）——积分-发放界限在真实数据上成立。

## 发现——以及一个诚实的负面结果

1. **SSM 通路输出是 delta 的甜点区，且遥遥领先。** 在
   真实权重上它可 4–45× delta 压缩；块中其他每个信号都是 1.0–1.3×。
   一个由 SSM 输出馈入的 matvec 在阈值 0.05 处做少 12× 的工作，
   代价是约 5 % 的每通道误差。

2. **SSM 本身无法被 delta 压缩——而这没关系。** 它是一个
   逐通道递归 `h_t = a·h_{t-1} + b·x_t`；每一步都依赖上一步，
   所以没有一步可被跳过。但它已经是 O(通道数)，不是
   瓶颈。它在 delta 推理中的角色是*慢信号生成器*：
   它是一个逐通道低通滤波器，所以其输出是网络中位置
   最相关的信号——这恰恰使得其下游的 matvec 对 delta 友好。
   此前 RESULTS 的建议（"把 delta 扩展到 SSM"）说对了一半：
   把它扩展到*消费* SSM 的那个 matvec，而非 SSM 递归本身。

3. **诚实的负面结果：post-norm `h`、conv 输出和注意力输出对 delta
   不友好（约 1.0–1.3×）。** LayerNorm 对每个位置重新归一化，因此 `h`
   在几乎每个通道上都移动；conv 与注意力输出确实逐位置变化。
   在注意力 Wq/Wk/Wv 投影（它们消费 `h`）上做 delta 推理
   几乎没有收益。别在那里部署它。

4. **更深的块比块 0 对 delta 更友好**（8 块平均 45×
   vs 块 0 在阈值 0.10 处 33×）——SSM 状态随深度升温，因此
   慢信号性质在网络更深处增强。

## 精炼后的建议

在**消费 SSM 通路输出的那些 matvec 层**上部署 delta 推理，
而非在注意力投影上，也非在 SSM 递归本身上。在真实的 944K
模型上，这是在阈值 0.05 处**实测 8–12× 的计算削减**
（约 5 % 每通道误差，有界），在阈值 0.10 处升至 33–45×。
把该阈值与 L11 状态范数监视器配对作为漂移看门狗。

## 质量成本 — 已测量（2026-05-20，`quality_real.py`）

早先的草稿保留了能量数字，因为阈值化误差的*质量*成本
尚未测量。现已测量。在消费信号 S 的 matvec 上做 delta 推理
等价于把一个积分-发放阈值化后的 S 喂给精确的 matvec；
所以我们对每个块的 SSM 输出做阈值化，精确运行真实 944K 模型
的其余部分，并测量交叉熵。

| 阈值 | SSM 路径加速 | Δ 困惑度 |
|---|---|---|
| 0.00 | 1.0× | +0.00%（精确——健全性检查） |
| 0.02 | 4.1× | −0.46%（在噪声范围内） |
| **0.05** | **12.6×** | **+0.57%** |
| 0.10 | 49× | +5.6% |
| 0.20 | 320× | +11.5%（崩溃） |

**该主张成立。** 在阈值 0.05 处，SSM 通路在训练好的模型上给出真实的
**12.6× 迭代次数削减，代价是 +0.57 % 困惑度**——一个可出货的
权衡。在 0.10 处它很激进（49× / +5.6 %）；在 0.20 处
它崩溃。这也反驳了"SSM 输出近乎常量、毫无贡献"的担忧：
若它毫无贡献，则在任何阈值下让它陈旧化都应是免费的——相反，
0.20 花费 +11.5 %，所以 SSM 输出确实重要，且模型确实容忍
*有界的*陈旧化。

## 剩余的诚实告诫

上面的质量数字是**预填充位置的交叉熵（prefill）**，而非
自回归生成。SSM 的低通性质应当会延续到
生成（它是一个递归滤波器），但生成步的测量尚未
运行。陈述 12.6×/+0.57 % 这个数字时请附上该告诫。
